About this Project

TechBlog is a portfolio project, it's source code can be found at: https://github.com/JonasMartins/project_b, it's goal is simulate an application based in a blog/social network, which uses React JS as front end and Node JS as back end, both written in Typescript and as SGBD was chose PostgreSQL. This project is similar to a social network, where any person with a valid email could create an account and interact with other users, keep conversations between them, create, react and comment on posts in your feed, improvements like OAuth and Email Provider, may be added in the future.

Authentication

The authentication used in the project uses the library express-session, that when a user successfully enters the system, informing his unique e-mail and the password, the server returns to the client a cookie with the session id that was successfully saved. There are some alternatives for store these sessions on the server, a common option is to use Redis ( example ). After the cookie is saved on the client with the session id and other information of the logged in user , every request sent to the api, it is checked if the id of the sent session by the request is saved in the redis store, so there is a user logged in, all application routes perform this validation.

File Upload

Allows the creation of posts with the attachment of multiple images, originally the images were hosted on Microsoft Azure, but in the development period for convenience, the blob files are now stored on the server's local disk, in the future when the project is put into production, the images most likely will back to be stored in the cloud, each user can also change their profile picture.

Notification System

Initially, some user interactions with the system can generate notifications, currently, the interactions that cause notifications are, reacting to a post, commenting on a post or reply to a post comment, where the affected person receives a new notification. In the example below a user reacts and comments on a post, this generates a notification to the post author, this is done using GraphQL Subscriptions, which uses the Web Socket protocol.
